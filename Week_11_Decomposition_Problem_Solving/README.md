# Week 11 — Decomposition & Problem Solving

This week focuses on computational thinking — breaking complex problems into
manageable sub-problems and solving each independently. All three tasks are larger
in scope than previous weeks and require careful planning before coding.

## Tasks

| File | Task | Key Concept |
|------|------|-------------|
| `01_timetable_scheduler.py` | Build a timetable scheduler that assigns lessons to slots without conflicts | Decomposition, constraint solving |
| `02_stats_toolkit.py` | A statistics toolkit providing mean, median, mode, range, and standard deviation | Modular design, algorithms |
| `03_grid_path_counter.py` | Count all valid paths through a grid from top-left to bottom-right | Recursion, problem decomposition |

## Approach
Before writing any code this week, sketch out the sub-problems on paper first.
Identify what each function needs to do, what it takes as input, and what it
returns. This is the decomposition step.
